# Root Privacy Policy

The repository-level privacy policy lives in `PRIVACY.md` at the project root.

## Why it is in the root

This project is open source and can be self-hosted. A root-level policy works better as the canonical project privacy reference than a website-only page because it can describe:

- extension-local storage behavior
- API request and voting data
- website cookie and analytics behavior
- the difference between official and self-hosted deployments

## Operator responsibilities

`PRIVACY.md` is not enough on its own for every deployment.

Anyone running a public instance should still review and publish deployment-specific details such as:

- legal entity or controller name
- contact email
- hosting and processor list
- retention periods
- international transfers
- any analytics, CDN, or reverse-proxy setup

## Maintenance

Update `PRIVACY.md` whenever the project changes how data is stored locally, what the API receives, or how the public website handles consent and analytics.
