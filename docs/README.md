# Documentation Index

This folder contains the detailed project documentation.

If you are new to the project, start here and then open the area you care about.

## User-Facing and Packaging Docs

- `client-extension.md`: how the current browser extension works
- `client-publishing.md`: how to build release packages for Chromium and Firefox
- `store-assets.md`: how to use the screenshot mockup in `example/` for storefront imagery

## Server and Deployment Docs

- `api.md`: API routes, payloads, and response behavior
- `server-deployment.md`: stack deployment, updates, backups, and operations

## Project Quality Docs

- `testing.md`: how tests are organized and run
- `security.md`: current injection hardening and remaining security work
- `privacy-policy.md`: explains the repository-level privacy policy and how operators should adapt it for real deployments

## Suggested Reading Order

For non-developers:

1. Return to the main `README.md`
2. Visit the project website
3. Read install instructions there

For contributors:

1. `client-extension.md` or `api.md`, depending on what you want to change
2. `server-deployment.md` if you need to run the stack locally
3. `testing.md` before opening a code change
